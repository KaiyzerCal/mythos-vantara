# Positioning

## One-Line

MAVIS is the AI chief of staff that remembers who you are, what you're building, and what happened yesterday.

## Category

Personal AI OS — not a productivity tool, not a chatbot, not a journal. A persistent, memory-aware operator layer that sits between you and everything you're working on.

## Differentiation

| Feature | Other AI tools | MAVIS |
|---|---|---|
| Session memory | Resets | Persistent across months |
| Context model | What you type now | Who you are over time |
| Persona | Generic assistant | Named, voiced, relationship-aware |
| Integration | Usually siloed | Google Workspace, Telegram, web |
| Interaction model | Q&A | Inbox/Outbox — she proactively surfaces what matters |

## Key Claims (that must be defensible)

1. MAVIS remembers. — True: semantic memory in Supabase, retrieved by topic.
2. MAVIS knows your current state. — True: energy, fatigue, floor, quests injected into every context.
3. MAVIS doesn't reset. — True: sessions archived, importance-scored, searchable.
4. MAVIS is yours, not generic. — True: system prompt, persona, voice, name all set by operator.

## Messaging Hierarchy

1. Memory persistence (most differentiating)
2. Knows your context now (energy, quests, arc)
3. Named, voiced, relationship-building persona
4. Multi-surface (web, Telegram, voice)
5. Gamified OS layer (CODEXOS)
